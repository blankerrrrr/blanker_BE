# 기능 명세서

## 1. 제품 개요

본 제품은 사용자가 웹을 탐색하는 동안 원치 않는 스포일러, 유해정보, 민감정보를 실시간으로 감지하고 차단하는 필터링 도구이다.

사용자는 관심 작품, 인물, 주제와 현재 진도를 기준으로 필터 프로필을 설정할 수 있다. 시스템은 웹페이지에 표시되는 텍스트, 이미지, 동적 로딩 콘텐츠를 분석해 위험도와 관련도를 판단하고, 차단 대상 콘텐츠를 자동으로 블러 처리한다. 차단된 콘텐츠는 나중에 보기 보관함에 저장할 수 있으며, 관심 주제와 관련된 일반 정보는 별도로 수집할 수 있다.

## 2. 목표

- 사용자가 원치 않는 스포일러와 민감 콘텐츠에 노출되지 않도록 한다.
- 웹페이지의 텍스트, 이미지, 동적 콘텐츠를 실시간으로 분석한다.
- 콘텐츠를 단순 차단하는 데서 끝내지 않고 차단 사유와 관련 주제를 사용자에게 제공한다.
- 차단된 콘텐츠를 나중에 확인할 수 있도록 보관한다.

## 3. 대상 사용자

- 스포일러를 피하면서 웹 콘텐츠를 탐색하고 싶은 사용자
- 특정 작품, 인물, 주제에 대한 노출 범위를 직접 관리하고 싶은 사용자
- 중요한 일정, 공지, 발매 정보를 놓치지 않고 모아보고 싶은 사용자

## 5. 기능 상세

### FEAT-01. 관심 대상 설정

- 사용자는 관심 작품, 인물, 주제를 등록, 수정, 삭제할 수 있다.
- 각 관심 대상은 이름, 별칭, 관련 키워드를 가질 수 있다.
- 시스템은 등록된 관심 대상을 콘텐츠 분석과 관련도 판단의 기준으로 사용한다.

### FEAT-02. 차단 유형 및 민감도 설정

- 사용자는 차단할 정보 유형을 선택할 수 있다.
- 기본 유형은 스포일러, 유해정보, 관심정보로 구분한다.
- 민감도는 낮음, 보통, 높음 단계로 설정한다.
- 민감도가 높을수록 불확실한 콘텐츠도 더 적극적으로 차단한다.

### FEAT-03. 웹 콘텐츠 분석

- 시스템은 현재 웹페이지의 텍스트와 이미지 콘텐츠를 분석한다.
- 텍스트는 제목, 본문, 댓글, 버튼, 링크 텍스트를 포함한다.
- 이미지는 대체 텍스트, 주변 문맥, 이미지 자체 분석 결과를 함께 고려한다.

### FEAT-04. 콘텐츠 분류

- 시스템은 분석 대상 콘텐츠를 스포일러, 유해정보, 관심정보, 일반정보로 분류한다.
- 하나의 콘텐츠는 여러 유형에 동시에 해당할 수 있다.
- 분류 결과는 차단 여부, 보관함 분류, 사용자 안내에 사용한다.

### FEAT-05. 동적 콘텐츠 감지

- 시스템은 무한 스크롤, 댓글 추가, 추천 피드 갱신 등 새롭게 로딩되는 콘텐츠를 감지한다.
- 새 콘텐츠가 DOM에 추가되면 기존 기준과 동일하게 분석한다.
- 사용자가 페이지를 이동하지 않아도 지속적으로 보호 상태를 유지한다.

### FEAT-06. 위험도 및 관련도 판단

- 시스템은 사용자 프로필과 콘텐츠의 관련도를 계산한다.
- 시스템은 콘텐츠의 노출 위험도를 계산한다.
- 차단 여부는 관련도, 위험도, 민감도 설정을 조합해 결정한다.

### FEAT-07. 자동 블러 처리

- 차단 대상 콘텐츠는 자동으로 블러 처리한다.
- 블러 처리된 콘텐츠는 원문을 직접 노출하지 않아야 한다.
- 사용자는 차단된 영역임을 인지할 수 있어야 한다.

### FEAT-08. 단위별 차단

- 시스템은 텍스트 문장, 이미지, 댓글, 게시글 카드, 페이지 영역 단위로 차단할 수 있다.
- 최소 차단 단위는 콘텐츠 의미를 유지하면서 과도한 화면 훼손을 줄이는 방향으로 결정한다.
- 이미지와 텍스트가 함께 위험한 경우 묶어서 차단할 수 있다.

### FEAT-09. 차단 사유 표시

- 사용자는 블러 처리된 콘텐츠의 차단 사유를 확인할 수 있다.
- 표시 정보는 차단 유형, 관련 주제, 위험도 수준을 포함한다.
- 차단 사유는 원문 스포일러나 유해 표현을 직접 노출하지 않는 방식으로 제공한다.

### FEAT-10. 차단 콘텐츠 저장

- 사용자는 차단된 콘텐츠를 나중에 보기 보관함에 저장할 수 있다.
- 저장 항목에는 콘텐츠 요약, 차단 유형, 관련 주제, 저장 시각을 포함한다.
- 저장된 콘텐츠는 사용자가 직접 삭제할 수 있다.

### FEAT-11. 발견 위치 기록

- 저장된 콘텐츠는 원본 URL을 기록한다.
- 가능하면 페이지 내 발견 위치 또는 DOM 기준 위치를 함께 기록한다.
- 원본 페이지 구조가 변경된 경우 위치 복원은 보장하지 않는다.

### FEAT-12. 관심 정보 자동 수집

- 시스템은 관심 주제와 관련된 일반 정보를 자동으로 수집한다.
- 수집 대상은 사용자가 등록한 관심 작품, 인물, 주제와 관련된 웹 콘텐츠이다.
- 수집된 정보는 원본 링크, 발견 시각, 관련 주제와 함께 저장한다.

### FEAT-13. 중복 콘텐츠 통합

- 시스템은 동일하거나 유사한 관심 정보가 여러 출처에서 발견되면 하나의 묶음으로 통합한다.
- 중복 판단 기준은 제목, 본문 핵심 문장, 원본 URL, 관련 주제, 발견 시각을 조합해 결정한다.
- 통합된 항목은 대표 콘텐츠와 추가 출처 목록을 함께 제공한다.
- 완전히 동일하지 않더라도 같은 공지, 같은 발매 정보, 같은 이슈로 판단되는 경우 중복 후보로 묶을 수 있다.

### FEAT-14. 로그인 및 회원가입

- 사용자는 이메일과 비밀번호로 회원가입할 수 있다.
- 사용자는 가입한 계정으로 로그인, 로그아웃할 수 있다.
- 로그인한 사용자는 필터 프로필, 관심 대상, 차단 설정, 보관함을 계정 단위로 관리할 수 있다.
- 회원가입 시 필수 약관 동의와 개인정보 처리 동의를 받아야 한다.
- 로그인 실패 시 계정 존재 여부나 보안 정보를 과도하게 노출하지 않는 오류 메시지를 제공한다.

## 6. 후속 기능 범위

MVP 이후 구현할 기능은 다음과 같다.

- 현재 진도 설정
- 여러 필터 프로필 생성 및 전환
- 콘텐츠별 일시 공개 또는 영구 허용
- 작품, 주제, 유형별 자동 분류
- 진도 달성 시 자동 잠금 해제
- 저장 콘텐츠 검색, 삭제, 읽음 처리
- 일정, 공지, 발매 정보 추출
- 스포일러 정보와 일반 정보 분리 고도화
- 일간, 주간 요약 제공
- 오탐, 누락 신고
- 특정 표현 허용 또는 차단
- 사용자 행동 기반 필터 조정
- 개인 맞춤형 탐지 정확도 개선
- 로컬 분석과 클라우드 분석 선택
- 민감한 입력 영역 분석 제외
- 저장 데이터 자동 삭제
- 사이트별 예외 설정
- 기기 간 설정 동기화

## 8. 사용자 흐름

### 8.1 기본 필터링 흐름

1. 사용자가 회원가입 또는 로그인을 완료한다.
2. 사용자가 관심 작품, 인물, 주제를 등록한다.
3. 사용자가 차단할 정보 유형과 민감도를 설정한다.
4. 사용자가 웹페이지를 탐색한다.
5. 시스템이 페이지의 텍스트와 이미지를 분석한다.
6. 시스템이 콘텐츠의 유형, 위험도, 관련도를 판단한다.
7. 차단 대상 콘텐츠를 블러 처리한다.
8. 사용자가 차단 사유를 확인하거나 보관함에 저장한다.

### 8.2 동적 콘텐츠 감지 흐름

1. 사용자가 피드, 댓글, 검색 결과 등 동적 페이지를 탐색한다.
2. 새 콘텐츠가 로딩된다.
3. 시스템이 새 콘텐츠를 감지한다.
4. 시스템이 새 콘텐츠를 분석하고 필요한 경우 블러 처리한다.

### 8.3 관심 정보 수집 흐름

1. 사용자가 관심 작품, 인물, 주제를 등록한다.
2. 시스템이 웹 탐색 중 관심 주제와 관련된 정보를 감지한다.
3. 시스템이 동일하거나 유사한 정보가 이미 수집되어 있는지 확인한다.
4. 중복으로 판단되면 기존 항목에 추가 출처로 통합한다.
5. 새로운 정보로 판단되면 관심 정보 피드에 새 항목으로 저장한다.

## 10. 정책 및 제약사항

- 사용자별 설정과 보관함 데이터는 인증된 계정에만 연결되어야 한다.
- 차단 사유는 원문 스포일러나 유해 표현을 그대로 노출하지 않아야 한다.
- 저장된 콘텐츠와 접근 기록은 삭제 가능해야 한다.
- 이미지 분석이 실패하더라도 텍스트 분석 기반 보호는 유지해야 한다.
- 동적 콘텐츠 분석은 페이지 사용성을 과도하게 저하하지 않아야 한다.

## 11. 비기능 요구사항

- 성능: 일반적인 웹페이지 탐색 흐름을 방해하지 않는 수준으로 분석을 수행해야 한다.
- 안정성: 분석 실패, 권한 부족, 네트워크 오류 시 사용자에게 최소한의 상태 피드백을 제공해야 한다.
- 개인정보: 로컬 분석 우선 구조를 고려하고, 클라우드 분석 사용 시 명확한 동의가 필요하다.
- 보안: 비밀번호는 평문으로 저장하지 않고, 인증 세션은 만료와 재발급 정책을 가져야 한다.
- 접근성: 블러 영역, 차단 사유, 보관함 UI는 키보드 조작과 스크린 리더 접근을 고려해야 한다.
- 확장성: 정보 유형, 민감도, 콘텐츠 분석 모델을 교체하거나 추가할 수 있어야 한다.

## 12. 성공 기준

- 사용자가 관심 대상과 차단 민감도를 설정할 수 있다.
- 사용자가 회원가입, 로그인, 로그아웃을 수행할 수 있다.
- 웹페이지의 텍스트와 이미지가 분석되고 위험 콘텐츠가 자동으로 블러 처리된다.
- 동적으로 로딩되는 콘텐츠도 동일하게 감지된다.
- 차단된 콘텐츠에는 차단 사유와 관련 주제가 표시된다.
- 차단된 콘텐츠를 보관함에 저장하고 원본 링크를 확인할 수 있다.
- 관심 주제 관련 정보가 자동 수집되고 중복 콘텐츠가 하나의 항목으로 통합된다.
